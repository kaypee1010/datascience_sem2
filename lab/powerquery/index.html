Practical 7 — Overview
Aim of this Practical
Import data from an external CSV file into Excel using Power Query, and perform systematic data cleaning and transformation steps using the Power Query Editor, then load the clean data back into Excel for analysis.

What is Power Query?

🔍 The Tool
Power Query is a built-in Excel feature (available from Excel 2016 onward) that lets you connect to external data sources — CSV files, websites, databases, JSON, XML — and clean, transform, and shape data before loading it into Excel.

It works without writing any formula. Every step you perform is recorded as a transformation step you can edit, reorder, or delete.
⚡ Why Use It?
Without Power Query, you'd manually find-replace, delete rows, and retype values — and redo it all next month.

With Power Query, your cleaning steps are saved as a recipe. When new data arrives, click Refresh — Power Query re-applies all steps automatically in seconds.

Part B — Data Cleaning

Step 1 — Promote First Row as Headers

📋 Action
  1. In the Power Query Editor, click the Home tab.
  2. Click "Use First Row as Headers" button (top-left area of ribbon).
  3. The first row (RegNo, StudentName, Department…) becomes the column headers.
  4. You'll see a new step appear in Applied Steps: PromotedHeaders
Why?
When CSV files are imported, Excel may treat the header row as data row 1. Promoting it tells Power Query these are column names, not data values.

Step 2 — Set Correct Data Types

🔢 Change Marks to Number
  1. Click the Marks column header to select it.
  2. On the Home tab → click "Data Type" dropdown → select Whole Number.
  3. A step ChangedType appears. Power Query will show errors for "500" — leave for now (we fix it in Outliers step).
🔡 Change Semester to Number
  1. Click the Semester column header.
  2. Data Type → Whole Number.
  3. Kabir's row shows Error because his Semester = "One" (text). We'll handle this next.

Step 3 — Replace Errors in Semester Column

  • 1
    Right-click the Semester column header → select "Replace Errors".
  • 2
    In the dialog box that appears, type 1 (since we know Kabir is in Semester 1). Click OK.
  • 3
    Kabir's Semester now shows 1 instead of Error.

Step 4 — Remove Rows with Missing Names

Decision
BCA009 has no Student Name. Since we cannot identify this student, we will remove this row. If the name were known, we'd fill it. For anonymous records, removal is the appropriate decision.
  • 1
    Click the dropdown arrow (▼) on the StudentName column header.
  • 2
    Uncheck (null) from the list → click OK. Rows with null/blank names are removed.
  • 3
    A step FilteredRows appears. You now have 9 rows (BCA009 is gone).

Step 5 — Fill Riya's Missing Marks with Average

🧮 Calculate Average Marks First
Valid marks (excluding 500 and blank): 78, 65, 82, 30, 76, 89, 55, 70
Average = (78+65+82+30+76+89+55+70) ÷ 8 = 545 ÷ 8 = 68.125 ≈ 68
In Power Query:

1. Click the Marks column dropdown (▼)
2. Uncheck (null) → this temporarily hides Riya's row for inspection
3. Instead — right-click the Marks column → "Replace Values"
4. Find: (leave blank / null) → Replace with: 68 → OK
5. Riya's marks are now filled with 68

Similarly fill Riya's Grade: Replace Values in Grade column → null → B

Step 6 — Trim Extra Spaces from StudentName

  • 1
    Click the StudentName column header to select it.
  • 2
    Go to the Transform tab in the ribbon.
  • 3
    Click "Format" dropdown → select "Trim". This removes leading and trailing spaces.
  • 4
    " Rahul Joshi " becomes "Rahul Joshi". Step TrimmedText added.

Step 7 — Standardize Text Case (Department & FeePaid)

📝 Department Column
  1. Click Department column → Transform tab → Format → UPPERCASE.
  2. Result: BCA / bca / Bca all become BCA.
📝 FeePaid Column
  1. Click FeePaid column → Transform tab → Format → Capitalize Each Word.
  2. Yes / yes / YES / NO / No → all become Yes / No.

Step 8 — Replace Outlier Value (Rahul's Marks = 500)

Why 500 is Wrong
Maximum possible marks = 100. A value of 500 is a data entry error. We replace it with the class average (68) — the same value used for missing marks — so both missing and erroneous values are treated consistently.
  • 1
    Click the Marks column → right-click → "Replace Values".
  • 2
    Find: 500 → Replace with: 68 → click OK.
  • 3
    Rahul's marks change from 500 to 68.

Part C — Transformation
Cleaning vs Transformation
Cleaning = fix existing data problems (remove nulls, fix formats, replace errors).
Transformation = reshape or add new information to the data (new columns, split columns, merge, filter, sort, group).

Transformation 1 — Add a "Result" Column

Goal
Add a new column called Result that shows "Pass" if Marks ≥ 40 and "Fail" if Marks < 40. This is a conditional calculated column.
  • 1
    In Power Query Editor, click the Add Column tab in the ribbon.
  • 2
    Click "Conditional Column". A dialog box opens.
  • 3
    Fill in:
    • Column Name: Result
    • If column: Marks → is greater than or equal to → Value: 40 → Output: Pass
    • Otherwise: Fail
  • 4
    Click OK. A new "Result" column appears. Arjun (30 marks) gets Fail, all others get Pass.

Transformation 2 — Add a "Performance Band" Column

  • 1
    Add Column → Conditional Column again. Name it: PerformanceBand.
  • 2
    Rules (add each as a condition row using the "+ Add Clause" button):
    • If Marks ≥ 75 → High
    • If Marks ≥ 50 → Medium
    • Otherwise → Low
  • 3
    Click OK. Each student is now categorized.

Transformation 3 — Filter to View Only Semester 1 Students

🔍 Filter Operation
  1. Click the dropdown (▼) on the Semester column.
  2. Uncheck 2 — only Semester 1 students remain.
  3. Note: This is for demonstration. In your final output, remove this filter (click the step in Applied Steps → delete it) so all students are included.

Transformation 4 — Sort by Marks (Descending)

  • 1

Transformation 5 — Split "RegNo" into Prefix and Number

Column Splitting
BCA001 can be split into "BCA" (Department prefix) and "001" (Student number). This is useful when you need to group by department from a mixed ID column.

3. Two new columns appear: RegNo.1 (= BCA) and RegNo.2 (= 001).
4. Rename them: double-click header → type Dept_Code and StudentNum.

Part D — M Language
What is M Language?
Every action you take in Power Query (clicking buttons, selecting options) is automatically written in a language called M (Power Query Formula Language). You can view and even edit this code directly.

How to View the M Code

  • 1
    In Power Query Editor → click View tab → tick "Advanced Editor".
  • 2
    A black window opens showing all your steps as M code. Each step = one line of code.
  • 3
    You can close this without changes, or edit the code directly for advanced transformations.

Understanding the M Code Structure

let // Step 1: Connect to source file Source = Csv.Document(File.Contents("C:\Users\YourName\Desktop\studentmarks.csv"), [Delimiter=",", Encoding=TextEncoding.Utf8]), // Step 2: First row becomes headers PromotedHeaders = Table.PromoteHeaders(Source, [PromoteAllScalars=true]), // Step 3: Set data types for each column ChangedType = Table.TransformColumnTypes(PromotedHeaders,{ {"Marks", Int64.Type}, {"Semester", Int64.Type}}), // Step 4: Replace errors in Semester (fixes "One" → 1) ReplacedErrors = Table.ReplaceErrorValues(ChangedType, {{"Semester", 1}}), // Step 5: Remove rows where StudentName is null FilteredRows = Table.SelectRows(ReplacedErrors, each [StudentName] <> null and [StudentName] <> ""), // Step 6: Replace missing Marks with 68 (average) ReplacedNullMarks = Table.ReplaceValue(FilteredRows, null, 68, Replacer.ReplaceValue, {"Marks"}), // Step 7: Trim leading/trailing spaces from StudentName TrimmedNames = Table.TransformColumns(ReplacedNullMarks, {{"StudentName", Text.Trim, type text}}), // Step 8: Uppercase the Department column UpperDept = Table.TransformColumns(TrimmedNames, {{"Department", Text.Upper, type text}}), // Step 9: Replace outlier Marks value 500 → 68 ReplacedOutlier = Table.ReplaceValue(UpperDept, 500, 68, Replacer.ReplaceValue, {"Marks"}), // Step 10: Add conditional Result column (Pass/Fail) AddedResult = Table.AddColumn(ReplacedOutlier, "Result", each if [Marks] >= 40 then "Pass" else "Fail") in AddedResult

Key M Language Syntax Rules

ConceptExampleMeaning

Step namesFilteredRows = …Each step has a name. The last name after in is the final output

Step 3 — Refreshing Data (The Power of Power Query)

🔄 Auto-Refresh Feature
This is the most powerful feature of Power Query:
When the examination department sends you next semester's CSV file (same format), simply:
1. Replace the old CSV file on your Desktop with the new one (keep the same filename)
2. In Excel → click any cell in the table → Data tab → "Refresh All"
3. Power Query re-runs ALL 9 cleaning and transformation steps automatically on the new data!

You never have to redo the work manually — the recipe is saved permanently.

🧠 Viva Preparation

Attempt each answer before revealing. These questions commonly appear in lab vivas for Power Query practicals.

What is Power Query and where is it found? Q1
Power Query is a built-in data connection and transformation tool in Microsoft Excel (and Power BI). It is found under the Data tab → Get & Transform Data group. It allows users to import data from external sources (CSV, database, web) and clean/transform it before loading into Excel — without writing formulas.
What is the difference between "Load" and "Transform Data" in Power Query? Q2
Load directly imports the raw data into Excel without any changes — the dirty data goes straight in.
Transform Data opens the Power Query Editor, where you can clean, fix, reshape and add columns before loading. Always choose Transform Data when the data has quality issues.
What are Applied Steps and why are they important? Q3
Applied Steps is the list in the right panel of Power Query Editor that records every transformation you perform. Each step is stored as a line of M code. They are important because: (1) you can click any step to see the data at that point, (2) you can delete or reorder steps, and (3) when you refresh the query on new data, all steps re-execute automatically — making the cleaning process repeatable and automated.
What is M Language in Power Query? Q4
M (Power Query Formula Language) is the programming language that Power Query uses internally. Every button click in the editor generates M code automatically. The code follows a let … in structure where each transformation is a named step. Advanced users can write or edit M code directly in the Advanced Editor for custom transformations not available through the UI.
How would you handle a missing value in Power Query? Q5
There are three approaches depending on the situation:

1. Remove the row — if the record is unusable without that value (e.g., missing ID or name): use column dropdown → uncheck null.
2. Replace with average/median — for numerical columns: right-click column → Replace Values → null → calculated average value.
3. Fill Down/Up — if values should be carried from the row above: Transform tab → Fill → Down.

The correct choice depends on the business context and how much data you can afford to lose.
What is the advantage of using Power Query over manual Excel cleaning? Q6
Key advantages:
1. Automation / Refresh — steps are saved; click Refresh to reapply them on new data automatically.
2. No formula clutter — transformations are done before loading; the Excel sheet stays clean.
3. Repeatable & Auditable — Applied Steps document every transformation, making the process transparent.
4. Multiple data sources — can connect to CSV, web, SQL, JSON simultaneously.
5. Non-destructive — original source file is never modified.
What happens if you click "Refresh All" in Excel after using Power Query? Q7
Excel reconnects to the original data source (the CSV file or database), re-reads all the data, and re-applies every step in the Applied Steps list from beginning to end. The resulting clean, transformed data is then reloaded into the Excel table. This means if you update the source file with new records, clicking Refresh automatically processes the new data with the same cleaning rules.

One-Line Definitions — Memorize These

Power Query
Excel's built-in ETL tool to import, clean, and transform data from external sources before loading into a sheet.
Applied Steps
The recorded list of every transformation action taken in Power Query, stored as M code and re-executable on refresh.
M Language
The formula language Power Query uses internally; follows a let…in structure with named transformation steps.
ETL
Extract, Transform, Load — the three-stage process Power Query performs: get data, clean it, put it in Excel.
Conditional Column
A new column added in Power Query based on if-then-else logic applied to existing column values.
Promote Headers
A step that converts the first data row of an imported file into column headers for the table.
Replace Errors
A Power Query feature that substitutes type-mismatch or parse errors in a column with a specified default value.
