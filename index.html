Part B — Data Cleaning

Step 1 — Handle Missing Values

Rule
Missing Marks → replace with AVERAGE of all other valid marks  |  Missing Study Hours → replace with MEDIAN
Use MEDIAN for hours because it is not affected by extreme values the way AVERAGE is.
🧮 CALCULATE — Riya's Marks
Valid marks (ignoring 500 & blank): 78, 65, 82, 76, 30, 89
=AVERAGE(78, 65, 82, 76, 30, 89)
Sum = 78 + 65 + 82 + 76 + 30 + 89 = 420
Count = 6
Average = 420 ÷ 6 = 70

✅ Replace Riya's blank marks with 70
🧮 CALCULATE — Ananya's Study Hours
Known study hours: 3, 4, 2, 1, 3, 0, 5
=MEDIAN(3, 4, 2, 1, 3, 0, 5)
Sorted: 0, 1, 2, 3, 3, 4, 5
Middle value (4th of 7) = 3

✅ Replace Ananya's blank hours with 3

Part C — Normalization
Why Normalize?
Marks go 0–100. Study hours go 0–5. These different scales make direct comparison unfair. Normalization brings everything to a 0–1 scale so we compare apples to apples.

The Formula

Normalized Value = (X − Min) / (Max − Min)
What each part means
X = the value you want to normalize

Min = lowest value in the column

Max = highest value in the column

Result is always between 0 (lowest) and 1 (highest)
Our values
Min Marks = 30 (Arjun)  |  Max Marks = 89 (Kavya)
So Max − Min = 89 − 30 = 59
🧮 LIVE CALCULATOR

Excel Formula to Type

=(G2 - MIN($G$2:$G$9)) / (MAX($G$2:$G$9) - MIN($G$2:$G$9))

Type this in a new column "Normalized Marks" (column H). The $ signs lock the range so the formula works when you drag it down.

Part E — Data Storytelling
What is Data Storytelling?
Converting raw numbers and charts into clear insights and real decisions. This is the most important skill — turning data into something a non-technical person can understand and act on.

Your 3 Key Insights (Write These in Your Record)

  • 1
    Study hours and marks are positively linked.
    Kavya (5 study hours) scored 89. Arjun (0 hours) scored only 30. Students who study more consistently score better. This suggests study time is a strong predictor of performance.
  • 2
    Attendance below 50% is linked to poor marks.
    Arjun (40% attendance) was the only student who failed (30 marks). Rahul (60%) also scored below average. Students with low attendance need extra support before exams.
  • 3
    Dirty data caused a 94-mark error in the class average.
    Before cleaning, average = 162.6 (impossible — out of 100). After cleaning, average = 68.75. This proves that unvalidated data can completely mislead decision-makers.

Your 2 Problems Identified

Problem 1 — Data Inconsistency
Gender and Payment columns used multiple formats for the same values (M, Male, F, Female, upi, UPI). This would break any grouping or counting analysis and produce incorrect reports.
Problem 2 — Outlier Distortion
Rahul's marks entry of 500 is a data entry error that inflated the entire class average by ~94 marks. No analyst would trust results from this uncleaned dataset.

Your 2 Decisions / Recommendations

Decision 1
Arrange extra coaching for low-attendance students.
Students with attendance below 60% (Arjun, Rahul) should receive mandatory extra classes before the next exam, based on the clear link between attendance and marks.
Decision 2
Implement data validation in the entry form.
Marks column should only accept values 0–100. Gender should use a dropdown list. This prevents dirty data from entering the system in the first place.

Why Is Normalization Important Here?

Marks (0–100), Study Hours (0–5), and Attendance % (0–100) are all on different scales. If we tried to compare them directly, a 1-hour difference in study time would look tiny compared to a 1-mark difference — even though 1 hour may be very significant.

Normalization puts all three on a 0–1 scale, so we can fairly compare which factor has the most influence on a student's performance. It also prevents one large-scaled variable from dominating in any statistical model.

What is dirty data? Q1
Dirty data is data that contains errors, missing values, inconsistent formats, or outliers — making it unreliable for analysis. Example: Gender column having Male, male, M all in the same dataset.
Why is data cleaning important? Q2
Because inaccurate data leads to wrong conclusions and poor decisions. In our dataset, one wrong entry (500) changed the class average by 94 marks. Cleaning improves data quality and makes analysis trustworthy.
What is normalization? Q3
Normalization scales data to a 0–1 range using the formula: (X − Min) / (Max − Min). It allows fair comparison between values that are on different scales (like marks out of 100 vs study hours 0–5).
What is the difference between mean and median? Q4
Mean = Sum ÷ Count. It is affected by extreme values (outliers).

Median = Middle value when data is sorted. It is NOT affected by outliers.

That is why we used median for Study Hours (in case of outliers) and mean for Marks (after removing the outlier).
Why are outliers dangerous? Q5
Outliers are extreme values that distort statistical results. In our dataset, Marks = 500 inflated the average from ~69 to ~163 — giving a completely false picture. Outliers can ruin averages, charts, and predictions if not handled.
What is data storytelling? Q6
Data storytelling is the process of converting raw data and analysis results into clear, meaningful insights and actionable decisions — using charts, summaries, and plain language that non-technical people can understand and act upon.
Why should data be standardized? Q7
Inconsistent formats like Male/male/M make grouping, filtering, and counting impossible. Excel and any other tool treats them as different values. Standardization ensures data can be correctly sorted, filtered, and analyzed — preventing wrong results in reports.

One-Line Definitions — Memorize These

Data Cleaning
Fixing errors, missing values, and inconsistencies to make data reliable.
Outlier
An extreme value far from the rest of the data that distorts analysis.
Normalization
Scaling all values to a 0–1 range using (X−Min)/(Max−Min).
Data Storytelling
Converting data analysis into clear insights and decisions for people to act on.
Mean
Sum of all values divided by the number of values. Affected by outliers.
Median
Middle value when data is arranged in order. Not affected by outliers.
